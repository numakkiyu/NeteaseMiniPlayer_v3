# 选择版本

NeteaseMiniPlayer v3 在同一个开源仓库中维护两条产品线。NMPv3 负责轻量嵌入，NMPv3+ 负责高级扩展。两者不是同一个包里的功能开关。

## 推荐选择

| 场景                                                   | 使用版本 | 说明                         |
| ------------------------------------------------------ | -------- | ---------------------------- |
| 在公开网页、博客、CMS 或静态站点中嵌入网易云播放器     | NMPv3    | 一个 JavaScript 文件即可运行 |
| 在 React、Vue3、Nuxt、Astro 或 Svelte 中输出基础播放器 | NMPv3    | 直接使用原生 `<nmp-player>`  |
| 需要本地 JSON、歌词文件、插件、皮肤或页面联动          | NMPv3+   | 这些能力由高级框架提供       |
| 需要普通 WordPress 短代码                              | NMPv3    | 前台只加载 `nmpv3.min.js`    |
| 需要 WordPress 后台、Gutenberg 区块或资源包构建        | NMPv3+   | 属于高级集成                 |

## NMPv3 的边界

NMPv3 的目标是保持可嵌入、可发布、可迁移：

- 默认浏览器部署只需要 `nmpv3.min.js`
- 不要求额外 CSS
- 不依赖 React、Vue、Lit、jQuery 或其他运行时库
- 音乐来源限定为网易云音乐及兼容 API 代理
- 保留 v2.5 的基础 UI、短代码和旧 DOM 兼容能力

NMPv3 可以通过属性、全局配置和 CSS 变量做基础配置。它不包含插件系统、完整皮肤系统、自定义音乐源或 HostBridge。

## NMPv3+ 的边界

NMPv3+ 面向需要扩展能力的集成者：

- 插件系统
- 皮肤系统
- 自定义音乐源
- 自定义歌词
- 宿主页面联动
- 框架适配器
- WordPress 与 PHP 高级集成
- CLI 自定义构建

NMPv3+ 默认从 NMPv3 的基础播放器界面开始。它的重点是扩展框架清晰，而不是把默认界面做得更复杂。

## 迁移建议

已有旧站点可以先迁移到 NMPv3，确认单曲、歌单、短代码和旧 DOM 兼容路径正常后，再评估是否需要 NMPv3+。

如果需求中出现“插件”“皮肤包”“本地音乐”“自定义歌词”“页面联动”或“框架适配器”，直接按 NMPv3+ 设计。
